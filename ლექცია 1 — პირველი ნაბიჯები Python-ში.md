მოგესალმებით ყველას და კეთილი იყოს თქვენი მობრძანება Python-ის ამ კურსზე!

მე ვარ [სახელი] და მინდა თავიდანვე გითხრათ: თუ ცხოვრებაში კოდის ერთი ხაზიც კი არ დაგიწერიათ, თქვენ ზუსტად ის ადამიანი ხართ, ვისთვისაც ეს კურსი შეიქმნა. შესაძლოა, ადრეც შეგიხედავთ კოდით შევსებული ეკრანისთვის და გიფიქრიათ, რომ ეს რაღაც გაუგებარი უცხო ენაა, ან იქნებ ერთხელ სცადეთ კიდეც სწავლა, მაგრამ ტექნიკურმა ტერმინოლოგიამ თავბრუ დაგახვიათ და ჩათვალეთ, რომ ეს თქვენი საქმე არ იყო.

თუ ოდესმე გქონიათ პროგრამირების შიში, ან გიფიქრიათ, რომ „მათემატიკური ტვინი“ არ გაქვთ, ნება მომეცით, ეს ეჭვი ახლავე გაგიფანტოთ: თქვენ სწორ ადგილას ხართ. პროგრამირება არ ნიშნავს გენიოსობას; ის ნიშნავს პრობლემის გადაჭრას და ფიქრის ახალი წესის სწავლას.

ეს კურსი შთაგონებულია ჰარვარდის CS50-ით — მსოფლიოში ერთ-ერთი ყველაზე ცნობილი და საყვარელი შესავალი კურსით კომპიუტერულ მეცნიერებაში. ჩვენი მიზანია, ავიღოთ იგივე მსოფლიო დონის ენერგია და სიმარტივე და გადმოვიტანოთ აქ. ჩვენ არ ვაპირებთ მხოლოდ თეორიებზე საუბარს ან სტატიკური სლაიდების ყურებას. ჩვენ ვაპირებთ დეველოპინგს და კოდის წერას.

ჩვენ ერთად, ნაბიჯ-ნაბიჯ გავივლით ყველა ფუნდამენტურ კონცეფციას. და ყველაზე მნშვნელოვანი ისაა, რომ კადრს მიღმა არანაირი „მაგია“ არ მოხდება. ყველაფერს, რასაც ავხსნი, პირდაპირ ჩემს ეკრანზე გაჩვენებთ, რათა ზუსტად დაინახოთ, როგორ იქცევა ცარიელი ფურცელი მომუშავე პროგრამად.

ასე რომ, ღრმად ჩაისუნთქეთ, დაივიწყეთ წარსულის იმედგაცრუებები და დავიწყოთ ეს მოგზაურობა. თქვენს პირველ პროგრამამდე სულ რამდენიმე წუთი გაშორებთ.

## hello.py

მე ვაპირებ გავხსნა VS Code — Visual Studio Code. თქვენ არ ხართ ვალდებული მაინცდამაინც ეს ინსტრუმენტი გამოიყენოთ. საბოლოო ჯამში, კოდი უბრალოდ ტექსტია, ასე რომ, ტექნიკურად მისი დაწერა Notepad-ში ან Google Docs-შიც კი შეგიძლიათ, თუ ძალიან მოინდომებთ. მაგრამ VS Code სპეციალურად ამისთვის არის შექმნილი, ის უფასოა, პოპულარულია და ჩვენ მას მთელი კურსის განმავლობაში გამოვიყენებთ.

ჩემი ეკრანის ქვედა ნაწილში შეამჩნევთ **ტერმინალის ფანჯარას** — ამას ეწოდება CLI (Command Line Interface), ანუ ბრძანების სტრიქონის ინტერფეისი. აქ არ არის აიკონები და ღილაკები, მხოლოდ ტექსტია. შესაძლოა, ეს უკან გადადგმულ ნაბიჯად მოგეჩვენოთ იმასთან შედარებით, რასაც მიჩვეული ხართ, მაგრამ მენდეთ — პროგრამისტები აქ ცხოვრობენ და მალე მიხვდებით, რატომაც.

ჩვენი პირველი ფაილის შესაქმნელად, პირდაპირ ტერმინალში ჩავწერ:

```
code hello.py
```

ეს გახსნის ახალ ფაილს სახელად `hello.py`. ბოლოში დაწერილი `.py` კომპიუტერს ეუბნება, რომ ეს არის Python-ის ფაილი.

ახლა კი, ყოველგვარი წინასწარი ცოდნის გარეშე, დავწეროთ ჩვენი პირველი პროგრამა:

Python

```
print("hello, world")
```

სულ ეს არის. ერთი ხაზი. მაშინაც კი, თუ არასდროს დაგიპროგრამებიათ, ალბათ ხვდებით, რას აკეთებს ეს — ის ეკრანზე დაბეჭდავს სიტყვებს: _hello, world_.

სხვათა შორის, ეს არის ყველაზე ცნობილი პირველი პროგრამა კომპიუტერული მეცნიერების ისტორიაში. თითქმის ყველა ენა, კურსი და სახელმძღვანელო აქედან იწყება. ჩვენც ასე ვიქცევით.

## პროგრამის გაშვება — CLI და Python-ის ინტერპრეტატორი

აი, რა ხდება ახლა: თქვენს ტელეფონსა თუ ლეპტოპში პროგრამებს ხატულაზე დაჭერით ან "ორმაგი კლიკით" უშვებთ. აქ ეს არ გვაქვს. ჩვენ ტერმინალში ვმუშაობთ, ამიტომ ბრძანება გვჭირდება.

დავბრუნდები ტერმინალში და ავკრეფ:

```
python hello.py
```

სანამ Enter-ს დავაჭერდე — ნება მომეცით აგიხსნათ, რა ხდება სინამდვილეში. კომპიუტერებს, საფუძვლების დონეზე, მხოლოდ ნულები და ერთები ესმით. მაგრამ სიტყვა `print`, ფრჩხილები და ეს ტექსტი? ეს ნამდვილად არ არის ნულები და ერთები. ამიტომ, ჩვენ გვჭირდება რაღაც, რაც ამას _თარგმნის_.

სწორედ ეს არის Python — არა მხოლოდ ენა, არამედ პროგრამა, რომელსაც **ინტერპრეტატორი** (interpreter) ეწოდება. როდესაც ვუშვებ ბრძანებას `python hello.py`, მე Python-ის ინტერპრეტატორს ვეუბნები: _აიღე ეს ფაილი, წაიკითხე თავიდან ბოლომდე და გადათარგმნე ისეთ რამედ, რაც კომპიუტერს რეალურად ესმის_.

კარგით, დავაჭიროთ Enter-ს.

```
hello, world
```

აი ისიც. თქვენი პირველი Python პროგრამა გაეშვა. თუ თქვენც იგივე ხაზი დაწერეთ და იგივე ბრძანება გაუშვით — გილოცავთ, თქვენ უკვე პროგრამისტი ხართ.

## ფუნქციები, არგუმენტები და გვერდითი ეფექტები

მოდით, ერთი ნაბიჯით უკან დავიხიოთ და დავფიქრდეთ, რა დავწერეთ სინამდვილეში.

Python

```
print("hello, world")
```

პროგრამირების ენების უმეტესობას მოჰყვება ჩაშენებული **ფუნქციების** ნაკრები. წარმოიდგინეთ ფუნქცია, როგორც ზმნა — მოქმედება, რომლის შესრულებაც კომპიუტერმა იცის. `print` ერთ-ერთი ასეთი ჩაშენებული ფუნქციაა. ეს არის მოქმედება: _გამოიტანე რაღაც ეკრანზე_.

მაგრამ `print` სულ ერთსა და იმავეს არ ბეჭდავს. ის ბეჭდავს _იმას, რასაც ეტყვით_. ხოლო ის, რასაც ფუნქციას გადასცემთ — ამ შემთხვევაში `"hello, world"` — არის **არგუმენტი**. არგუმენტი არის ფუნქციისთვის მიწოდებული ინფორმაცია (input), რომელიც გავლენას ახდენს მის მოქმედებაზე.

როდესაც `print` ეშვება და ჩვენ ეკრანზე ტექსტს ვხედავთ — ამას **გვერდითი ეფექტი** (side effect) ეწოდება. გვერდითი ეფექტი არის ის, რასაც ფუნქცია თვალსაჩინოდ _აკეთებს_ — მოცემულ შემთხვევაში, ტექსტის ჩვენება ეკრანზე. ყველა ფუნქციას არ აქვს ხილული გვერდითი ეფექტი, მაგრამ `print`-ს ნამდვილად აქვს. ეს მისი მთელი დანიშნულებაა.

შევაჯამოთ:

- **ფუნქცია** → ზმნა, მოქმედება (`print`)
    
- **არგუმენტი** → ინფორმაცია, რომელსაც აწვდით (`"hello, world"`)
    
- **გვერდითი ეფექტი** → რა ხდება შედეგად (ტექსტი ეკრანზე)

## ბაგები და დებაგინგი (Debugging)

სიმართლე უნდა გითხრათ: მე ახლა შეცდომა არ დამიშვია, რაც რეალურ ცხოვრებაში თითქმის არასდროს ხდება. როდესაც კოდის წერას სწავლობთ — და სიმართლე ითქვას, წლების შემდეგაც კი — თქვენ აუცილებლად დაუშვებთ შეცდომებს. პროგრამირებაში ამას **ბაგებს** (bugs) ვუწოდებთ.

მოდით, ერთი შეცდომა განზრახ დავუშვათ. წავშლი დამხურავ ფრჩხილს ჩვენი ხაზის ბოლოში:

Python

```
print("hello, world"
```

თითქმის სწორად გამოიყურება, არა? მხოლოდ ერთი სიმბოლო აკლია. მაგრამ ნახეთ, რა მოხდება, როცა მის გაშვებას ვცდი:

```
SyntaxError: '(' was never closed
```

**სინტაქსური შეცდომა** (SyntaxError) ნიშნავს, რომ დაწერეთ რაღაც, რაც ენას არ ესმის. კომპიუტერი ყველაფერს _სიტყვასიტყვით_ აღიქვამს — თითოეულ ფრჩხილს, ბრჭყალს და მძიმეს მნიშვნელობა აქვს. აქ "მიახლოებითი" ვერსიები არ მოსულა.

კარგი ამბავი ისაა, რომ ეს შეცდომის შეტყობინება რეალურად საკმაოდ დამხმარეა. ის ზუსტად გვეუბნება, რა არის არასწორი. ასე ყოველთვის არ იქნება — ხანდახან შეცდომის შეტყობინებები დამაბნეველი და გაუგებარია, და ეს ნორმალურია. შეცდომების კითხვა თავისთავად უნარია და დროთა განმავლობაში ამაში დაიხვეწებით.

მოდით, გამოვასწოროთ: დავაბრუნოთ ფრჩხილი და ისევ გავუშვათ:

```
hello, world
```

მშვენიერია. დავბრუნდით მწყობრში.

კარგი კითხვაა. ტექნიკურად, Word-ი გაძლევთ ტექსტის აკრეფის საშუალებას და კოდიც ტექსტია — ასე რომ, _ტექნიკურად_ კი. მაგრამ გთხოვთ, არ ქნათ ეს. Word-ი ამატებს ფორმატირებას, მუქ ტექსტს, სპეციალურ უხილავ სიმბოლოებს — რისი გაგებაც Python-ს არ შეუძლია. VS Code არის **უბრალო ტექსტის რედაქტორი** (plain text editor), რაც ზუსტად ისაა, რაც ჩვენ გვჭირდება. მას ასევე აქვს ისეთი სასარგებლო ფუნქციები, როგორიცაა კოდის გაფერადება — შეამჩნიეთ, რომ `print` ლურჯია, ხოლო ტექსტი სხვა ფერის. ამას **სინტაქსის გამოკვეთა** (syntax highlighting) ეწოდება და ის კოდის სწრაფად წაკითხვაში გეხმარებათ.

_სტუდენტის კითხვა: "შეგვიძლია პროგრამა ტერმინალის გარეშე გავუშვათ?"_

ამ ეტაპზე, ამ კონფიგურაციით — არა. ჩვენ ტერმინალს ვიყენებთ, რადგან ის გვაძლევს ყველაზე დიდ კონტროლს და სწავლის საუკეთესო გზაა. საბოლოოდ, თქვენ ნამდვილად შეძლებთ შექმნათ პროგრამები გრაფიკული ინტერფეისით, სადაც ნებისმიერს შეეძლება ღილაკებზე დაჭერა. მაგრამ ჯერჯერობით ტერმინალი ჩვენი სახლია.

## VS Code და IDE-ები

სანამ გავაგრძელებდეთ — ორიოდე სიტყვა იმ ხელსაწყოზე, რომელსაც ვიყენებთ. VS Code არის ის, რასაც **IDE**-ს უწოდებენ — Integrated Development Environment (განვითარების ინტეგრირებული გარემო). ის მხოლოდ კოდის საწერი ადგილი არ არის. მას აქვს ჩაშენებული ტერმინალი, სინტაქსის გამოკვეთა, ავტომატური დასრულება (შესაძლოა შეამჩნიეთ, რომ მან ფრჩხილი ავტომატურად დახურა) და ხელსაწყოები ბაგების საპოვნელად.

არსებობს ბევრი სხვა IDE. შეგიძლიათ გამოიყენოთ ის, რაც მოგწონთ. ჩვენ VS Code-ს ვიყენებთ, რადგან ის უფასოა, ძლიერია და მისი გაშვება ბრაუზერშიც კი შეგიძლიათ, არაფრის დაინსტალირების გარეშე.

კარგით — ჩვენი პროგრამა "მიესალმა" _სამყაროს_. მაგრამ რა მოხდება, თუ გვინდა, რომ ის კონკრეტულ _ადამიანს_ მიესალმოს? სწორედ აქეთკენ მივდივართ შემდეგ ნაწილში.
